Semicolons

Syntax

Description

Semicolons ; separate independent message chains that execute in parallel. Each chain runs independently — failure in one does not affect others.

Syntax

chain1 ; chain2 ; chain3

Example

<!-- Update two different receivers in parallel -->
<button sender="content get:apply: /page inner; sidebar get:apply: /sidebar inner">
  Load Both
</button>

<!-- Mix pipes and semicolons -->
<button sender="a get: /x | a apply: inner; b get:apply: /y text">
  Complex
</button>

Notes

  • Each chain is a separate sequence of piped messages
  • All chains start simultaneously via Promise.all()
  • Chains are fully independent — no value passing between them
  • Useful for updating multiple parts of the page at once
