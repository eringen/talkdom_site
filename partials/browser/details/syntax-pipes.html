Pipes

Syntax

Description

The pipe character | chains messages sequentially. The return value of each step is passed as the first argument to the next step.

Syntax

step1 | step2 | step3

Example

<!-- GET returns response text, piped as content to apply: -->
<button sender="content get: /page.html | content apply: inner">
  Load Page
</button>

<!-- Cross-receiver piping: fetch into one, apply to another -->
<button sender="api get: /data | sidebar apply: append">
  Append to Sidebar
</button>

How It Works

  • Steps execute left to right, waiting for each Promise to resolve
  • The resolved value becomes the first argument of the next step
  • Each step can target a different receiver
  • If any step rejects (error or cancel), the chain stops
