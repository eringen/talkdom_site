Keyword Messages

Syntax

Description

talkDOM uses Smalltalk keyword message syntax. A message starts with the receiver name, followed by keyword:argument pairs. Keywords always end with a colon. Multiple keywords form a compound selector.

Syntax

receiver keyword: arg
receiver keyword: arg keyword: arg

Parsing Rules

  • First token is always the receiver name
  • Tokens ending with : are keywords
  • Everything between keywords fills arguments
  • Keywords are joined to form the selector (e.g., get:apply:)

Examples

<!-- Single keyword -->
content get: /page.html
→ receiver: "content", selector: "get:", args: ["/page.html"]

<!-- Compound keywords -->
content get:apply: /page.html inner
→ receiver: "content", selector: "get:apply:", args: ["/page.html", "inner"]

<!-- Multi-word argument -->
dialog confirm: Are you sure you want to delete?
→ receiver: "dialog", selector: "confirm:", args: ["Are you sure you want to delete?"]
