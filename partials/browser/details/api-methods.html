talkDOM.methods

Programmatic API

Description

Register custom methods that can be used in sender messages. Methods receive the target element and parsed arguments. The method name must end with a colon.

Signature

talkDOM.methods["name:"] = function(el, arg1, arg2, ...) { ... }

Examples

// Toggle a CSS class
talkDOM.methods["toggle:"] = function(el, cls) {
  el.classList.toggle(cls);
};
// Usage: <button sender="menu toggle: hidden">Toggle</button>

// Add a CSS class
talkDOM.methods["addClass:"] = function(el, cls) {
  el.classList.add(cls);
};

// Remove element with animation
talkDOM.methods["fadeOut:"] = function(el, duration) {
  el.style.transition = "opacity " + duration;
  el.style.opacity = "0";
  setTimeout(function() { el.remove(); }, parseFloat(duration) * 1000);
};

// Multi-keyword method
talkDOM.methods["set:to:"] = function(el, attr, value) {
  el.setAttribute(attr, value);
};

Notes

  • Method name must end with : (Smalltalk convention)
  • Multi-keyword methods use compound selectors (e.g., "set:to:")
  • Return a Promise to enable piping and proper lifecycle events
  • The el parameter is the receiver element
  • Can override built-in methods (use with caution)
