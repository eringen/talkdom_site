confirm:

Control Method

Description

Shows a browser confirmation dialog. If the user clicks Cancel, the message chain is aborted by rejecting the Promise with "cancelled".

Syntax

receiver confirm: <message text>

Example

<!-- Confirm before deleting -->
<button sender="item confirm: Are you sure? | item delete:apply: /api/item/1 outer">
  Delete Item
</button>

Notes

  • Place at the beginning of a pipe chain to gate destructive operations
  • Uses the native confirm() dialog
  • If cancelled, subsequent piped steps are skipped entirely
  • The message text is the full argument after confirm:
