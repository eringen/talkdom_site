WebSocket Events

WebSocket

Description

WebSocket lifecycle events fire on all receiver elements subscribed to a URL and bubble up the DOM. Incoming messages also fire the standard talkdom:done event.

Events

talkdom:ws:open

Connection opened. Detail: { url }

talkdom:ws:close

Connection closed. Detail: { url, code, reason }

talkdom:ws:error

Connection error. Detail: { url }

Example

document.addEventListener("talkdom:ws:open", function (e) {
  console.log("connected to", e.detail.url);
});

document.addEventListener("talkdom:ws:close", function (e) {
  console.log("disconnected from", e.detail.url, e.detail.code);
});

Notes

  • Events bubble, so you can listen at document level
  • Fired on all receivers subscribed to the URL, not just the first
  • The standard talkdom:done event also fires after content is applied from a WebSocket message
