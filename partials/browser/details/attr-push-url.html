push-url

Attribute

Description

Updates the browser URL after a sender dispatches its message. Enables SPA-style navigation with working back/forward buttons.

Syntax

<a sender="..." push-url="/page">Link</a>

<!-- Empty value: uses first message arg as URL -->
<button sender="content get:apply: /page inner" push-url>Load</button>

Example

<nav>
  <a href="/docs"
     sender="content get:apply: /partials/docs.html inner"
     push-url="/docs">
    Docs
  </a>
</nav>

<main receiver="content">...</main>

Notes

  • Uses history.pushState() under the hood
  • Back/forward navigation replays the sender message via popstate event
  • Only pushes if the URL differs from the current path
  • The history state stores the full sender message string for replay
