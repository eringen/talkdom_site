ws:

WebSocket

Description

Declares a WebSocket connection on a receiver element. The server pushes content to the receiver in real time. Requires the talkdom-ws.min.js plugin.

Syntax

<div receiver="name ws: <ws-url>"></div>

Server Message Formats

JSON with receiver routing:

{"receiver": "feed", "content": "<p>New</p>", "op": "append"}

Raw talkDOM message syntax:

feed apply: Updated! text

Example

<!-- Real-time feed -->
<div receiver="feed ws: ws://localhost:3000/updates"></div>

<!-- Multiple receivers sharing one connection -->
<div receiver="messages ws: ws://localhost:3000/live"></div>
<div receiver="presence ws: ws://localhost:3000/live"></div>

Notes

  • Multiple receivers on the same URL share a single WebSocket connection
  • Auto-reconnects with exponential backoff (1s–30s, ±25% jitter)
  • Reconnection stops when all receivers for a URL are removed from the DOM
  • Dynamically added receivers are detected via MutationObserver
  • Default max 16 connections (configurable via talkDOM.ws.maxConnections)
